测试用例说明:

本测试用例，主要针对"caja环境 + 第三方代码（这里指设计师或者其他使用caja的第三方js代码） + 店铺前端规范” 进行测试。

测试用例运行方式简介：

  1. 宿主页面创建 "describe, expect,it" api, 模拟jasmine 语法， 暴露给caja环境
  2. 测试用例代码经过caja 编译后返回，然后执行
  3. describe，expect，it 方法会在页面创建 友好的界面来说明测试用例。
  4. api 含义参考 jasmine，具体测试用例直接看下面的测试结果

测试标准主要依据:

wiki规范

测试范围包括：

  1. JS 语言基础
    -- js语言层面，包括数组，字符串，日期, 正则表达式等基础对象和方法，逻辑语言
  2. dom 基础
    -- document，node等和浏览器相关的方法
  3. KISSY API
    -- 官方暴露给第三方js 所能使用的API接口。包括 DOM，Event,UA, Anim，Node,IO接口
  4. 安全测试
    1. 店铺html,css 规范相关的安全校验
    2. caja js 官方提供的攻击类型（全局对象方法访问, 禁止使用的全局方法:(window, document.cookie,document.document.location等)
    3. 官方提供的api安全校验 （IO接口的白名单校验，DOM.query(x)获取范围限定等）
  5. 性能测试
    -- '性能测试' 测试原生js和淘宝提供的api的性能
